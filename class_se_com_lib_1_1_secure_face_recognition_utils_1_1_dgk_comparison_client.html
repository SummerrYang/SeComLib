Paillier::Ciphertext SeComLib::SecureFaceRecognitionUtils::DgkComparisonClient::ComputeLambda ( const std::deque< Dgk::Ciphertext > &  e) const

Computes \( [\lambda] \).

Parameters
evector containing \( l + 1 \) DGK encryptions
Returns
\( \lambda \)
Warning
Protocol 4.10 from Martin Franz' Master Thesis returns \( [\lambda] = [0] \) in case zeros are detected, but it should be the other way around, since detecting a [0] means that \( 2 \hat{r} > 2 \hat{d} + 1 \), which implies an underflow

If the input contains at least one encrypted zero, then \( [\lambda] = [-2^l] \) and we can stop scanning the rest of the values

If the input contains no encrypted zeros \( [\lambda] = [0] \) ( \( 2 \hat{r} < 2 \hat{d} + 1 \) and no underflow has occured)
