\( [a] \leq [b] ? [1] : [0] \)

Compute \( [z] = [2^l + a - b] = [2^l] [a] [b]^{-1} \)

The MSB of \( z \), \( z_l \), is the result of the comparison \( (z_l = 0 \Rightarrow a < b)) \) To extract \( z_l \), we need to compute \( z_l = 2^{-l} (z - (z \pmod {2^l})) \)

\( (z \pmod {2^l}) \) needs to be computed interactively

Additively blind \( [z] \): \( [d] = [z + r] = [z] [r] \)

Re-randomize [d]

Ask the client to compute \( [-(d \pmod {2^l})] \)

Since \( d \equiv z + r \pmod {2^l} \Rightarrow z \pmod {2^l} = ((d \pmod {2^l}) - (r \pmod {2^l})) \pmod {2^l} \) Compute \( [-\tilde{z}] = [-(d \pmod {2^l}) + (r \pmod {2^l})] = [-(d \pmod {2^l})] [r \pmod {2^l}] \)

Because \( \tilde{z} \) is computed \( \pmod n \) instead of \( \pmod {2^l} \), in case of underflows, we need to add \( 2^l \) to get the right results. If \( d \pmod {2^l} \geq r \pmod 2^l \), \( z \pmod {2^l} = \tilde{z} \) Else, an underflow occurs, so we need to add \( 2^l \) to \( \tilde{z} \) We need to compare \( \hat{d} = d \pmod {2^l} \) (held by the client) and \( \hat{r} = r \pmod {2^l} \) (held by the server) interactively and store the result in \( \lambda \) Once the server obtains \( [\lambda] \), we can compute \( [z \pmod {2^l}] = [\tilde{z} + \lambda 2^l] = [\tilde{z}] [\lambda]^{2^l} \)

Choose random \( s \in {0, 1} \)
